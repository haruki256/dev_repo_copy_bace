# メモリーチェック手順

ユーザーから「メモリーチェック」と依頼があったときに実行する。

## 目的

memory/ 配下のファイルを棚卸しし、古い情報の削除・統合・移動を行う。
memory/ は git 管理外で気軽にファイルを作れる分、肥大化しやすいため定期的な整理が必要。

## 手順

### 1. 現状確認
- `memory/MEMORY.md` を読む
- `memory/hot/` と `memory/cold/` のファイル一覧を取得する
- MEMORY.md に載っているがファイルが存在しないポインタがないか確認する
- ファイルは存在するが MEMORY.md に載っていないものがないか確認する

### 2. 各ファイルの鮮度チェック
hot/ 配下の各ファイルについて：
- 記載されている情報が現在のコードと一致しているか確認する（ファイルパス、関数名、仕様など）
- 矛盾があればメモリを更新する（コードが正）
- 案件完了済みの情報は `hot/` → `cold/` に移動する
- もう不要な情報は削除する

### 3. MEMORY.md の更新
- ファイルの追加・削除・移動を反映する
- 200行以下に維持されているか確認する
- アクセスパターン別のグルーピングが適切か確認する

### 4. 報告
以下をユーザーに報告する：
- 削除したファイルと理由
- 移動したファイル（hot → cold）
- 更新した内容
- 現在のファイル数（hot / cold）
