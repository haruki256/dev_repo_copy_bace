# システム関連図

## このフォルダについて
メインシステムと他システム・サービスとの関連をまとめるフォルダ。

- `RELATION_MAP.md` — システム全体の関連図と連携一覧
- 各システムの詳細ファイルを必要に応じて追加（例：`[SYSTEM_NAME].md`）

---

## フォルダ構成ルール

- **メインシステムの全体像** は `doc/system/SYSTEM_OVERVIEW.md` に記載する（このフォルダ外）
- **このフォルダ** は、メインシステムと他システムの「関連」にフォーカスする
- 関連するシステムが増えた場合は、**各システムごとにファイルを作成** する

```text
doc/system/relations/
├── README.md              ← 本ファイル（フォルダ説明）
├── RELATION_MAP.md        ← 全体の関連図・連携一覧
├── [SystemA].md           ← 外部システムAの説明・連携方式
├── [SystemB].md           ← 外部システムBの説明・連携方式
└── ...
```

---

## 複数リポジトリ構成の方針

> **通常は1システム = 1リポジトリ**。フロント・バック分離等で複数リポジトリに分かれる場合は以下の方針に従う。

### フォルダを分ける場合のルール

1. **各リポジトリに `AI_CONTEXT.md` を持たせる**
   - 各リポジトリは独立したプロジェクトとして扱い、このテンプレート構成を適用する
   - リポジトリごとに `AI_CONTEXT.md` → `doc/` の構成を維持する

2. **`doc/system/relations/` は全リポジトリに配置する**
   - 同じ内容（関連図・リポジトリ一覧）を各リポジトリの `doc/system/relations/` に置く
   - どのリポジトリから見ても全体の関連がわかるようにする

3. **リポジトリ間の依存方向を明記する**
   - どのリポジトリがどのリポジトリに依存するかを `RELATION_MAP.md` に記録する

### 複数リポジトリ構成例

```text
[組織/チーム]/
├── [project]-frontend/          # フロントエンドリポジトリ
│   ├── AI_CONTEXT.md
│   ├── doc/
│   │   ├── system/
│   │   │   ├── SYSTEM_OVERVIEW.md
│   │   │   └── relations/       ← 全体の関連図（共通内容）
│   │   └── ...
│   └── src/
│
├── [project]-backend/           # バックエンドリポジトリ
│   ├── AI_CONTEXT.md
│   ├── doc/
│   │   ├── system/
│   │   │   ├── SYSTEM_OVERVIEW.md
│   │   │   └── relations/       ← 全体の関連図（共通内容）
│   │   └── ...
│   └── src/
│
└── [project]-shared/            # 共有ライブラリ（必要な場合のみ）
    ├── AI_CONTEXT.md
    └── ...
```

### リポジトリ分割の判断基準

| 状況 | 推奨 |
|------|------|
| サーバーサイドレンダリング中心 | 分割しない（1リポジトリ）|
| SPA + API構成 | モノレポ or マルチレポを検討 |
| モバイルアプリ + API | 分割推奨 |
| 共有ライブラリが必要 | 別リポジトリとして切り出し |

※ 詳細は `doc/guide/STAGE_GUIDE.md` の Stage 3 および `doc/guide/ARCHITECTURE_GUIDE.md` も参照。
