## プロジェクト全体のディレクトリ構造 (Stage 1 ベース)

基本構成として、単一の `src/` ディレクトリ内で入力（エンドポイント）または画面ごとにパッケージを分割するモノリシックな構成を想定しています。
※ フロントエンドとバックエンドが分離した構成（Stage 3）を取る場合は、それぞれのリポジトリまたはディレクトリ内で同様の設計思想を適用します。

```text
[Project Root]/
├── src/                          # アプリケーションのルートディレクトリ
│   ├── app/                      # エンドポイント・画面モジュール（APIの受付等）
│   │   ├── [Endpoint/Screen 1]/  #   機能1のロジック・エンドポイント
│   │   ├── [Endpoint/Screen 2]/  #   機能2のロジック・エンドポイント
│   │   └── settings/             #   設定関連
│   ├── common/                   # 副作用のない純粋なユーティリティ
│   │   └── utils/
│   └── shared/                   # 共通ライブラリ・共有リソース
│       ├── config/               #   設定管理
│       ├── persistence/          #   DB アクセス層 (DB使用時)
│       └── [Other Shared Dir]/
│
├── doc/                          # ドキュメント
│   ├── system/                   #   システム構成（このフォルダ）
│   ├── usage/                    #   ユーザー向けガイド
│   ├── guide/                    #   開発ガイド
│   └── ...
└── [Config Files]                # パッケージの設定ファイル等
```

## 基本アーキテクチャ

### エンドポイント・画面モジュール（app/）

各ユースケースの入り口、ビジネスロジック、データ構造を定義します。

| モジュール | 主な処理 | 役割 |
|-----------|------------|------|
| `app/[Endpoint 1]/` | [API Endpoints / Component] | [Endpoint 1 のロジック] |
| `app/[Endpoint 2]/` | [API Endpoints / Component] | [Endpoint 2 のロジック] |
| `app/settings/` | `get_settings`, `save_settings` | 設定管理 |

### 共通ライブラリ（shared/）

app/ から複数モジュールにまたがって使われる共通処理を定義します。

| モジュール | 役割 | 詳細 |
|-----------|------|------|
| `shared/persistence/` | データベース層 | [DB 種別] アクセス、エンティティ定義 |
| `shared/config/` | 設定管理 | ユーザー設定の読み書き |
| `shared/[Domain]/` | ドメイン共通機能 | 複数モジュール横断で用いるドメイン処理 |

### データの流れ（モジュール内）

```text
ユーザー操作 / 外部からのリクエスト
    ↓
Entrypoint (Controller / Handler)
    ↓
Business Logic (Service)
    ↓ [必要に応じて]
Database / External API (Shared / Persistence)
    ↓
レスポンスを受け取って状態・画面を更新（APIレスポンスの返却）
```
