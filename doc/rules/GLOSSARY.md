# 用語集・ユビキタス言語

プロジェクトで使用するユビキタス言語（ドメイン独自の共通言語）を定義します。
AI・開発者は、コードのコメント・コミットメッセージ・ドキュメントにおいて、この用語集に従って言葉を統一すること。

## 用語集の管理ルール
- **ファイル分割禁止**: 用語は必ずこの `doc/rules/GLOSSARY.md` に一元管理し、ファイルを分割したり別の場所に用語集を作らないこと。
- **グループ分けの維持**: 新しい用語を追加する際は、必ず適切な「カテゴリ（グループ）」に分類して追記すること。

## `::UBIQ` タグとの連動について
**UBIQ** は **Ubiquitous Language（ユビキタス言語）** の略称です。
本用語集で定義された「UBIQ」列の英語文字列は、ソースコード内の影響範囲特定タグ `::UBIQ` にそのまま使用されます。
長い用語であっても UBIQ は簡潔な英語名になっているため、ソースコードに埋め込む際や grep 検索時に便利です。
例: `// ::UBIQ Domain1, Domain2`
詳細は `CODING_RULES.md` を参照してください。

## 命名規則

### バックエンド（[Backend Directory]/）
- フレームワークと言語の標準的な命名規則（Snake Case, Pascal Case, Camel Case 等）に従う。

### フロントエンド（[Frontend Directory]/）
- フレームワークと言語の標準的な命名規則に従う。
- コンポーネント、ストア、ユーティリティなどを分かりやすく区別できる命名を心がける。
