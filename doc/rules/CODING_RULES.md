## 全般ルール

## `::UBIQ` ユビキタス言語タグ

### 目的
パッケージ単位（エンドポイント・画面）の分割を超えて、**特定のドメインの概念（ユビキタス言語）がどのソースコードに結びついているか** を特定しやすくするため、ソースコード内に影響タグを埋め込む。

### 形式
```
::UBIQ 用語1, 用語2, ...
```

- 記述する用語は、必ず `doc/rules/GLOSSARY.md` に定義されたユビキタス言語と一致させること。

### 配置ルール
- **各言語共通**: 各ファイルの先頭付近に、その言語のコメント構文（`//`, `<!-- -->`, `#` など）で記述する。
- 新規ファイル作成時に、そのソースが関与するユビキタス言語をタグとして付与する。
- 複数の概念が関与する場合はカンマ区切りで複数付与する。

### 例: 
```typescript
// ::UBIQ [Domain1], [Domain2]
// [FeatureName]: 〇〇の処理

function doSomething() {
// ...
}
```

### 使い方（AI・開発者向け）
```bash
# 「[Domain1]」というユビキタス言語に関するファイルをパッケージ横断で一覧化する
grep -r "::UBIQ.*[Domain1]" src/
```

- パッケージ境界を超えて返す場合は明確な型や構造体を使用する
- 想定外と想定内のエラーを区別して扱う

- 状態管理、UIの関心事、型定義などは使用技術に寄せて定義する。
