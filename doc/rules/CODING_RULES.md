## `::UBIQ` ユビキタス言語タグ

### 目的
パッケージ単位（エンドポイント・画面）の分割を超えて、**特定のドメインの概念（ユビキタス言語）がどのソースコードに結びついているか** を特定しやすくするため、ソースコード内に影響タグを埋め込む。

### 形式
```
::UBIQ[用語名]
```

- **1行に1つ**の用語を書く（カンマ区切りは使わない）
- 用語は角括弧 `[]` で囲む
- 記述する用語は、必ず `doc/rules/GLOSSARY.md` に定義されたユビキタス言語と一致させること

### 配置ルール

ファイルの先頭に以下の順序で記述する：

```
① ::UBIQ タグ（1行1つ）
② ファイル説明コメント（1行）
③ 空行
④ コード本体（import / use 等）
```

- `::UBIQ` タグは**ファイルの最上部**に置き、ファイル説明よりも**前**に書く
- `::UBIQ` ブロックとコード本体の間には**空行を1行**入れる
- 新規ファイル作成時に、そのソースが関与するユビキタス言語をタグとして付与する

### 上位概念・子概念のタグ付けルール

子概念に該当するソースコードを作成・編集する際は、**必ず「上位概念のタグ」と「子概念のタグ」の両方を併記**します。
これにより、上位概念のタグで完全一致検索（`-F`）した際に、関連する子概念のファイルもすべて影響範囲として抽出できるようになります。
階層関係（どれが親か）の定義は `doc/rules/GLOSSARY.md` に従います。

### 例

```typescript
// ::UBIQ[Domain1]           <- 上位概念（影響範囲検索のために必須）
// ::UBIQ[Domain1Detail]     <- 子概念（詳細な絞り込み用）
// [FeatureName]: 〇〇の処理

function doSomething() {
  // ...
}
```

```python
# ::UBIQ[Domain1]            <- 上位概念
# ::UBIQ[Domain1Detail]      <- 子概念
# 〇〇の処理

import os
```

```html
<!-- ::UBIQ[Domain1] -->       <!-- 上位概念 -->
<!-- ::UBIQ[Domain1Detail] --> <!-- 子概念 -->
<!-- 〇〇コンポーネント -->

<div>...</div>
```

### 検索方法（AI・開発者向け）

`-F` は固定文字列モード。`[]` を正規表現として解釈させない。

```bash
# 単一検索
grep -rnF "::UBIQ[Domain1]" src/

# AND検索（Domain1 かつ Domain2 を両方含むファイル）
grep -rlF "::UBIQ[Domain1]" src/ | xargs grep -lF "::UBIQ[Domain2]"

# OR検索（Domain1 または Domain2 を含むファイル）
grep -rnF -e "::UBIQ[Domain1]" -e "::UBIQ[Domain2]" src/
```
