# 必読ガイド

AI および開発者がプロジェクトに参加する際、どのような順序でドキュメントを読み進めるべきかをまとめます。

## 必読順序（最優先）
以下のファイルは、開発に着手する前に**必ず**目を通してください。

1. **`AI_CONTEXT.md`** (ルート直下)
   - プロジェクトの全体像と各種ルールへのポインタです。ここで前提条件を入れます。
2. **`doc/guide/必読ガイド.md`** (本ドキュメント)
   - どこから読み始めるかの道しるべです。
3. **`doc/rules/GLOSSARY.md`**
   - ユビキタス言語の定義です。表記ゆれを防ぐために必ず確認します。
4. **`doc/rules/PACKAGE_RULES.md`**
   - ファイルをどこに配置し、各パッケージがどのような役割を持つか（`app/`, `shared/`, `common/` 等）を定めています。
5. **`doc/rules/CODING_RULES.md`**
   - `::UBIQ`タグなどのコーディング規約です。

## 開発フェーズごとに必要なドキュメント

実際の開発作業において、シチュエーションに応じて以下のドキュメントを参照・更新してください。

### 新規画面の追加・要件定義を行う時:
- **`doc/AI/flow/00_flow.md`** : どのようなステップで進めるかの大枠
- 各ステップ詳細: `doc/AI/flow/step/` 配下
- 各種テンプレート: `doc/AI/flow/template/` 配下

### 実装に入り、パッケージ構成をいじる時:
- **`doc/design/PACKAGE_STRUCTURE.md`** : 現在のファイルツリー構造。ファイル追加時はここも更新します。

### アーキテクチャの変更や共通化を判断する時:
- **`doc/guide/ARCHITECTURE_GUIDE.md`** : アーキテクチャ選定の思想
- **`doc/guide/STAGE_GUIDE.md`** : プロジェクト規模に応じたステージごとの対処方針（モノリシックから分離していく等）

### システムの全体像を掴む時:
- **`doc/system/SYSTEM_OVERVIEW.md`** : プロジェクトの概要
- **`doc/system/ARCHITECTURE.md`** : 構成図
- **`doc/system/TECH_STACK.md`** : 利用技術・ライブラリ一覧

### アプリのセットアップ・使い方を知る時:
- **`doc/usage/SETUP_GUIDE.md`** : 環境構築手順
- **`doc/usage/USER_GUIDE.md`** : アプリの各種機能の使い方

### DB設計を確認・更新する時:
- **`doc/design/DB_DESIGN.md`** : テーブル一覧・ER図・カラム定義・インデックス設計

### どのファイルをいつ更新するか迷った時:
- **`doc/guide/UPDATE_TIMING.md`** : 全ドキュメントの更新タイミングマトリクス

### 複数システム・リポジトリの関連を把握する時:
- **`doc/system/relations/RELATION_MAP.md`** : システム関連図・連携一覧
- **`doc/system/relations/README.md`** : フォルダ構成ルール・リポジトリ分割方針
