# ファイル更新タイミングマトリクス

## このファイルについて
各ドキュメントファイルが「いつ更新が必要になるか」を一覧でまとめたもの。
新規参画者やAIが「今のタスクでどのファイルを確認・更新すべきか」を素早く判断できるようにする。

---

## タスク別の早見表

### 初回プロジェクト立ち上げ時
すべてのファイルのプレースホルダーを埋める。特に重要なのは：
1. `AI_CONTEXT.md` — プロジェクト名・概要・技術スタックを記入
2. `SYSTEM_OVERVIEW.md` — システムの全体像を記述
3. `GLOSSARY.md` — 初期のドメイン用語を定義
4. `DB_DESIGN.md` — テーブル設計を記述
5. `PACKAGE_STRUCTURE.md` — 初期ファイル構成を記述

### 新しい機能を追加する時
1. `PACKAGE_STRUCTURE.md` — 新規ファイルをツリーに追加
2. `GLOSSARY.md` — 新ドメイン用語があれば追加
3. `DB_DESIGN.md` — 新テーブル・カラムがあれば追加
4. `USER_GUIDE.md` — ユーザー向け機能の説明を追加

### パッケージ構成を大きく変える時
1. `STAGE_GUIDE.md` — 現在のStageを確認
2. `ARCHITECTURE_GUIDE.md` — 変更方針を確認
3. `PACKAGE_STRUCTURE.md` — 構成を更新
4. `AI_CONTEXT.md` — 必要に応じてプロジェクト構成セクションを更新

---

## 凡例

- ✅ 必須：このタイミングで必ず確認・更新が必要
- 🔄 条件付き：該当する変更があった場合に更新
- ─：通常は更新不要
