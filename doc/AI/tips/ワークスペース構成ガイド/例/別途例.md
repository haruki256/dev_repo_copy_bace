【人間向けメモ：守られやすいルールの書き方】
 
- 基本は否定形より肯定形で書く。
  悪い例:
  - 推測しない
  良い例:
  - 必要情報が不足している場合は確認する
 
- ただし、事故りやすい行為や明確な禁止事項は、否定形でも明示する。
  例:
  - 固有名詞を推測で補完しない
  - 指定されていない形式を勝手に追加しない
 
- ルールは、守ったかどうか判定できる形で書く。
  悪い例:
  - 丁寧に書く
  - 適切に対応する
  - 必要なら詳しく書く
  良い例:
  - 指定された出力形式がある場合は、その形式に厳密に従う
  - 不足情報がある場合は、補完せず確認する
  - 禁止事項に該当する内容は出力しない
 
- タイミングは具体的に書く。
  例:
  - 作業開始時に確認する
  - 実行中は各ステップで適用する
  - 最終出力直前に再確認する
 
- ルールは短く書き、1ルール1意味を基本にする。
 
- 実行時に本当に必要なルールだけを入れる。
  目安は3〜7個程度。多すぎると全部薄くなる。
 
- ルールと参考情報を混ぜない。
  ルール、背景説明、補足、参考例は分離する。
 
- 競合時の動きを書く。
  例:
  - ルールと依頼が衝突する場合は、その衝突を明示してから進める
 
- 情報不足時の動きを書く。
  例:
  - 必要情報が不足している場合は、推測せず確認する
 
悪い例:
1. Be careful.
2. Respect the context.
3. Be concise.
 
良い例:
1. If a required output format is specified, follow it exactly.
2. If required information is missing, ask for clarification instead of inferring.
3. If the request conflicts with an applicable rule, explicitly state the conflict before proceeding.
