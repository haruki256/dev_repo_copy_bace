# 2. 設計書作成（DESIGN.md）

* 目的：REQUIREMENTS.mdを基に、実装の全体像を把握する
* 行動：REQUIREMENTS.mdを基に、DESIGN.mdをAIに作成してもらう
* 作成物：DESIGN.md（設計書）
  * 再定義：実装目的／ゴール／要件／技術選定の確定
  * 影響範囲：既存機能への影響、呼び出し元など
  * ファイル構成：変更・新規作成ファイル一覧
* 人間の役割：設計書レビュー
* 補足：技術的な可否や案の比較はAIとこの段階までに完了させる。
* 実装手段が確定できない場合には、AIに対象手段の簡易実装をさせ、その結果を基に実装手段を確定させる。

資料作成のための情報が足りない場合などは、ユーザーに適宜質問を行い、必要な情報を収集してください。
資料作成後はチェック項目を共有し、チェックを促してください。

**DESIGN.md（テンプレ）**

```md
# 設計書
## 実装目的
## ゴール
## 要件
### 対象の振る舞い
### 制約
## 使用手段・技術選定
* 使用手段（ライブラリ、API等）は必要か？
    * その技術のIN-OUTは？
    * 使用条件や制約、注意点は何か？
* 利用技術は何か？
## 影響範囲の分析
* 変更により影響を受ける既存機能はどこか
* 呼び出し元はどこか
* 既存仕様との整合性は？
## ファイル構成
### 変更・新規作成ファイルの一覧
```
